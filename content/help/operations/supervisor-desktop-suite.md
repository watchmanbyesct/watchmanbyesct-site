The **supervisor desktop** (`/`) is the primary control surface for running a security program.

### Core areas you can run from desktop

- **Operations:** command center, incidents, dispatch, sync center
- **Workforce:** scheduling workspace, timekeeping, DAR/reports
- **Field administration:** patrol, checkpoints, post orders, handover, inspections, maintenance
- **Management:** personnel, assignments, sites, vehicles, report center, communications, messages
- **Advanced/optional:** live map, automation, intelligence, incident analysis, billing, services, audit log, admin

### Display

The layout is **desktop-first** with a grouped sidebar. On a **tablet or small laptop**, the menu usually tucks into a **drawer** you open from the edge of the screen.

### Scheduling and guard availability

Scheduling uses the **scheduling workspace** view. Guard availability appears there as availability indicators and override context used during planning.

### Access notes

Some modules require both role entitlement and tenant-module enablement. Platform-level pages (for example audit log and admin) are restricted to platform-owner access.
